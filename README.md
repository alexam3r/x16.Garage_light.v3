# Гаражный свет на ESP8266

Прошивка для управления светом в гараже на базе ESP8266 (Wemos D1 mini).
Среда разработки — PlatformIO + Arduino framework, язык C++.

Устройство подключается к Wi-Fi и MQTT-брокеру, управляет двумя группами света
(основное освещение и декоративные лампы Эдисона) в двух взаимоисключающих
режимах, реагирует на PIR-датчик движения и раз в минуту публикует JSON-статус
с показаниями датчика температуры/влажности.

## Аппаратная часть

- **Контроллер:** Wemos D1 mini (ESP8266).
- **Питание:** БП 220VAC → 5VDC.
- **Реле:** плата с двумя электромеханическими реле (основное + Edison).
- **Датчики:** AM2320 (температура/влажность, I²C) и PIR (движение).

### Распиновка

| Пин | GPIO | Функция |
| --- | ---- | -------- |
| D5  | GPIO14 | Основное освещение (реле 1) |
| D6  | GPIO12 | Декоративные лампы Эдисона (реле 2) |
| D7  | GPIO13 | PIR — датчик движения |
| D1  | GPIO5  | I²C SDA → AM2320 |
| D2  | GPIO4  | I²C SCL → AM2320 |

AM2320 питается от 3.3V; PIR и реле-плата — по своей логике (обычно 5V с общим GND).

## Возможности

- **PIR-автоматика:** при обнаружении движения свет включается на 10 минут;
  повторное срабатывание обнуляет таймер. Триггер — по фронту LOW→HIGH.
- **MQTT-управление:** команда `light/set ON` включает свет на 1 час.
- **Два режима света** (`mode`): `main` (D5) и `edison` (D6), взаимоисключающие.
  Команда `mode/set edison` при выключенном свете включает свет в режиме edison
  на 1 час; при горящем свете — переключает активную лампу, не трогая таймер.
- **Сброс при выключении:** когда свет гаснет (таймер истёк или `light/set OFF`),
  режим возвращается на `main`. Реакцией на PIR это не управляет — она задаётся
  только `motion/set` (с авто-возвратом через 6 часов).
- **Отключаемая реакция на PIR:** `motion/set OFF` — PIR перестаёт включать свет,
  но факт движения продолжает публиковаться в `motion/state`. Состояние самой
  реакции (вкл/выкл) отдаётся отдельным retain-топиком `motion/enabled` — чтобы
  клиенты (например Home Assistant) получали его мгновенно, без ожидания статуса.
  Реакция сама включается обратно через 6 часов (`MOTION_AUTO_ENABLE_MS`) — сброс
  к дефолту на случай, если ушёл из гаража и забыл вернуть; `motion/set ON`
  отменяет авто-возврат.
- **JSON-статус** в топике `status`: состояние света, режим, факт движения,
  включена ли реакция на PIR, температура, влажность, uptime, уровень Wi-Fi сигнала.
  Публикуется раз в минуту, а при изменении любого параметра состояния (свет,
  режим, движение, реакция на PIR) — немедленно, с обнулением минутного таймера
  (чтобы не слать статус дважды подряд).
- **Watchdog:** если Wi-Fi и MQTT одновременно отсутствуют 10 минут —
  устройство перезагружается (`ESP.restart()`). Аппаратный watchdog core ESP8266
  защищает от зависаний.

## Настройка

1. Скопируйте `src/Secrets.h.sample` в `src/Secrets.h` и заполните реальные значения:
   - `WIFI_SSID`, `WIFI_PASSWORD` — Wi-Fi сеть;
   - `MQTT_HOST` — IP/hostname брокера;
   - `MQTT_USER`, `MQTT_PASSWORD` — креды брокера (пусто = без auth);
   - `MQTT_CLIENT_ID` — уникальный ID клиента.
2. При необходимости отредактируйте `src/Config.h`:
   - `RELAY_ACTIVE_HIGH` — полярность реле (`1` = HIGH включает; если реле
     срабатывает «наоборот» — поставьте `0`);
   - `LIGHT_TIMEOUT_PIR_MS` / `LIGHT_TIMEOUT_MQTT_MS` — длительности таймера;
   - `CFG_ENABLE_AM2320` — `0`, если датчик не подключён (temp/humidity = `null`).
3. `Secrets.h` исключён из git (`.gitignore`) — не коммитить креды.

## MQTT-топики

Корень дерева — `garage/light`.

| Топик | Направление | Payload | Retain | Назначение |
| --- | --- | --- | --- | --- |
| `garage/light/availability` | out | `online`/`offline` | да | LWT + статус на коннекте |
| `garage/light/light/set` | in | `ON`/`OFF` | — | команда включения/выключения |
| `garage/light/light/state` | out | `ON`/`OFF` | да | текущее состояние света |
| `garage/light/mode/set` | in | `main`/`edison` | — | команда выбора режима |
| `garage/light/mode/state` | out | `main`/`edison` | да | текущий режим |
| `garage/light/motion/set` | in | `ON`/`OFF` | — | включить/выключить реакцию на PIR |
| `garage/light/motion/state` | out | `ON`/`OFF` | нет | факт обнаружения движения (по фронту) |
| `garage/light/motion/enabled` | out | `ON`/`OFF` | да | состояние реакции на PIR (вкл/выкл) |
| `garage/light/status` | out | JSON | да | статус устройства (раз в минуту + при изменении состояния) |

### Пример JSON статуса

```json
{"light":"on","mode":"main","motion":"off","motion_enabled":"on","temperature":23.4,"humidity":55.0,"uptime":3600,"wifi_rssi":-67}
```

- `motion` — факт обнаружения движения PIR прямо сейчас (`on`/`off`).
- `motion_enabled` — включена ли реакция на PIR (`on`/`off`; `off` после `motion/set OFF`,
  сам вернётся в `on` через 6 часов).

При недоступном датчике: `"temperature":null,"humidity":null`.

## Проверка через mosquitto

```bash
# подписка на все топики устройства
mosquitto_sub -h <broker> -t 'garage/light/#' -v

# включить свет
mosquitto_pub -h <broker> -t 'garage/light/light/set' -m 'ON'

# выключить свет
mosquitto_pub -h <broker> -t 'garage/light/light/set' -m 'OFF'

# включить режим Edison (при выключенном свете — включит свет в edison на 1 час)
mosquitto_pub -h <broker> -t 'garage/light/mode/set' -m 'edison'

# отключить реакцию на PIR (сама включится через 6 часов)
mosquitto_pub -h <broker> -t 'garage/light/motion/set' -m 'OFF'

# вернуть реакцию на PIR вручную (отменяет авто-возврат)
mosquitto_pub -h <broker> -t 'garage/light/motion/set' -m 'ON'
```

## Логика таймера и режимов

- **Один таймер выключения** + флаг источника (`PIR` или `MQTT`).
- PIR при выключенном свете → включение на `LIGHT_TIMEOUT_PIR_MS` (10 мин), источник `PIR`.
- PIR при включённом свете → обнуление таймера на длительность **текущего**
  источника (PIR → 10 мин, MQTT → 1 час). Источник переключается только явной
  MQTT-командой (`light/set ON` или `mode/set`); PIR не переподчиняет MQTT-сеанс.
- `light/set ON` → источник `MQTT`, таймер 1 час.
- `mode/set` при горящем свете → переключение лампы, таймер не меняется.
- `mode/set` при выключенном свете → включение в выбранном режиме на 1 час.
- Выключение (таймер или `light/set OFF`) → режим `main`; реакция на PIR не меняется.
- `motion/set OFF` → реакция на PIR выключена; через `MOTION_AUTO_ENABLE_MS` (6 ч)
  включается сама. `motion/set ON` включает сразу и отменяет авто-возврат.

## Troubleshooting

- **`temperature`/`humidity` = `null`** — датчик AM2320 не отвечает. Проверьте
  подключение SDA/SCL (D1/D2), питание 3.3V, подтяжки I²C (4.7kΩ к 3.3V, если
  на модуле их нет). Для отладки логики без датчика поставьте `CFG_ENABLE_AM2320=0`.
- **Реле щёлкает «наоборот»** (ON выключает свет) — инвертируйте `RELAY_ACTIVE_HIGH`
  в `Config.h`.
- **PIR не срабатывает** — попробуйте `PIR_PIN_MODE INPUT_PULLUP` в `Config.h`
  (если датчик с открытым коллектором); проверьте питание PIR.
- **MQTT не подключается** — смотрите `mqtt.state()` в Serial-логе: `-2` = не удалось
  подключиться, `-4` = неверный логин/пароль, `2` = ID клиента занят.
- **Устройство перезагружается** — если Wi-Fi/MQTT отсутствуют 10 минут, это
  срабатывает connectivity-watchdog (нормально); если чаще — проверьте питание
  (brownout от индуктивных выбросов реле) и стабильность сети.
